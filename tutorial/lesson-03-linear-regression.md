## 2. 背景与原理介绍

线性回归是监督学习的最小闭环。虽然它很简单，但神经网络训练最核心的逻辑已经都在这里了：

1. 参数化模型：`y_hat = Xw + b`
2. 用损失函数衡量预测和真值的差距
3. 用梯度下降更新参数

如果你能不依赖任何框架把线性回归写通，后面所有更复杂的模型都只是“把模型函数换复杂了”。

## 3. 详细操作过程

### 步骤 1：自己生成一份合成数据

在 `practice/lesson03/linear_regression_scratch.py` 中先生成数据：

1. 设定真实参数 `w_true` 和 `b_true`
2. 随机生成输入特征 `X`
3. 用线性关系生成标签 `y`
4. 加入一点高斯噪声

这样你训练完以后，就能直接检查学出的参数是否接近真值。

### 步骤 2：初始化参数

手动初始化：

1. `w` 随机小数
2. `b` 初始化为 0

请你先不要纠结高级初始化，线性回归用最简单版本就足够。

### 步骤 3：写模型函数

实现：

```python
def linear_regression(X, w, b):
    return X @ w + b
```

注意输出维度是否和标签对齐。

### 步骤 4：写损失函数

这里使用均方误差：

```python
def mse_loss(y_hat, y):
    return ((y_hat - y) ** 2).mean() / 2
```

你可以思考为什么很多教材会多写一个 `/2`。原因是后面求导时更简洁。

### 步骤 5：手写梯度

请自己把下面两个梯度推出来：

1. `dL/dw`
2. `dL/db`

推完后在代码里实现。建议先写 batch 版，再验证维度。

### 步骤 6：实现 SGD 更新

每一轮训练都做：

1. 前向传播得到 `y_hat`
2. 计算损失
3. 计算梯度
4. 更新 `w` 和 `b`
5. 记录当前 loss

建议先用全量数据写通，再改成 mini-batch。

### 步骤 7：画图和验证

本课一定要做两个验证：

1. 画出 loss 曲线，看是否下降
2. 打印训练后参数，看是否接近 `w_true` 和 `b_true`

如果 loss 不下降，优先排查：

1. 学习率是否过大
2. 梯度公式是否写错
3. 维度是否广播错了

## 4. 本课检查点

1. 线性回归为什么是监督学习的最小范例？
2. 为什么损失下降不一定意味着梯度没问题，但损失完全不降通常说明有 bug？
3. 学习率太大和太小分别会出现什么现象？
4. 你的参数最终是否接近真实参数？
