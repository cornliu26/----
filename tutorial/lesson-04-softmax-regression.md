## 2. 背景与原理介绍

回归问题预测的是连续值，而分类问题预测的是“属于哪一类”。对于多分类任务，模型输出通常不是一个数，而是每个类别的打分。

Softmax 的作用，是把这些打分变成一个“和为 1 的概率分布”；交叉熵的作用，是衡量预测分布和真实分布之间的差距。二者搭配，构成了多分类中最经典的模型组合。

## 3. 详细操作过程

### 步骤 1：准备一个小型分类数据集

如果你想减少环境阻力，可以先选以下任一方式：

1. 人工生成 3 类二维点
2. 使用鸢尾花数据集
3. 后续再升级到 Fashion-MNIST

第一次实现建议先用二维点集，因为更容易画图和调试。

### 步骤 2：写线性分类器

模型形式：

`logits = X @ W + b`

这里的输出不是一个数，而是一个长度为类别数的向量。

### 步骤 3：实现稳定版 Softmax

直接指数化容易数值爆炸，所以请先减去每行最大值：

```python
shifted = logits - logits.max(axis=1, keepdims=True)
probs = np.exp(shifted) / np.exp(shifted).sum(axis=1, keepdims=True)
```

这一步非常重要，它会在后面很多模型中反复出现。

### 步骤 4：实现交叉熵损失

假设标签是整数类别编号，请你自己完成：

1. 取出每个样本真实类别对应的预测概率
2. 对概率取对数
3. 取负号并求平均

同时思考：为什么真实类别概率越接近 1，损失越小？

### 步骤 5：手写梯度并更新参数

这一课你可以直接使用 Softmax 回归的标准结论，也可以自己推导。推荐你至少完成下面两个动作：

1. 写出 `logits` 对参数 `W`、`b` 的梯度更新
2. 用训练集 accuracy 观察模型是否在学习

### 步骤 6：加入评估指标

新建 `metrics.py`，至少写一个函数：

```python
def accuracy(y_pred, y_true):
    ...
```

确保你能统计：

1. 训练准确率
2. 验证准确率
3. 每轮 loss

### 步骤 7：解释结果

如果 loss 下降但 accuracy 不升，常见原因有：

1. 数据类别分布有问题
2. 学习率设置不合适
3. 代码里的标签处理错位
4. Softmax 或交叉熵实现有 bug

## 4. 本课检查点

1. 为什么分类输出要用 Softmax，而不是直接输出一个最大值？
2. 为什么交叉熵会惩罚“对错误类别过于自信”的预测？
3. Softmax 里减去最大值为什么不改变结果？
4. 多分类模型里参数矩阵 `W` 的形状应该怎么确定？
