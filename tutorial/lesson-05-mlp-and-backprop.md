# 第 5 课：多层感知机与反向传播

## 1. 本课定位

- 对应 D2L：
  - `chapter_multilayer-perceptrons/mlp.md`
  - `chapter_multilayer-perceptrons/mlp-scratch.md`
  - `chapter_multilayer-perceptrons/backprop.md`
- 建议时长：3 到 4 小时
- 课时目标：
  - 理解为什么线性层叠加仍然是线性
  - 理解激活函数的作用
  - 手写两层 MLP 的前向和反向传播

## 2. 背景与原理介绍

只靠线性模型，很多真实任务都无法拟合。原因是多个线性变换连在一起，本质上仍然只是一个线性变换。要让模型表达更复杂的非线性关系，就必须引入激活函数，比如 ReLU、sigmoid、tanh。

多层感知机是“线性层 + 激活函数 + 线性层”的最经典结构，而反向传播则负责把最后的损失梯度，一层层传回去，告诉每一层参数应该怎么改。

## 3. 详细操作过程

### 步骤 1：先把网络结构画出来

请手画或用文字写出：

1. 输入层维度
2. 隐藏层维度
3. 输出层维度
4. 每一层参数的形状

例如：

`X -> Linear(784, 256) -> ReLU -> Linear(256, 10) -> Softmax`

### 步骤 2：先实现前向传播

在 `practice/lesson05/mlp_scratch.py` 中至少写出：

1. `linear_forward`
2. `relu`
3. `softmax`
4. `cross_entropy`

建议在每一层都打印一次张量形状，确保维度没错。

### 步骤 3：手推反向传播

本课最关键的是你要明确：

1. 损失对输出层 logits 的梯度
2. 输出层参数的梯度
3. 梯度如何乘以上一层权重继续往回传
4. ReLU 在反向传播中如何“截断”梯度

请至少自己写一版纸面推导，不要直接抄答案。

### 步骤 4：实现反向传播

推荐你为每一层缓存前向时需要的中间结果，例如：

1. 输入 `X`
2. 线性层输出 `Z`
3. 激活后输出 `A`

因为反向传播时要用这些中间量。

### 步骤 5：做梯度检查

本课非常适合做一次参数梯度检查。随机挑几个参数元素：

1. 用你的手写反向传播得到梯度
2. 用数值差分近似梯度
3. 比较误差

如果这里对不上，就先不要继续训练。

### 步骤 6：训练模型并观察现象

请记录：

1. 训练 loss
2. 训练 accuracy
3. 验证 accuracy

你会发现 MLP 比前一课的线性分类器表达能力更强，但也更容易训练不稳定。

## 4. 本课检查点

1. 为什么“线性层 + 线性层”不能提升表达能力？
2. ReLU 的优势是什么？
3. 反向传播为什么需要缓存中间结果？
4. 如果某一层梯度始终为 0，可能是什么原因？
