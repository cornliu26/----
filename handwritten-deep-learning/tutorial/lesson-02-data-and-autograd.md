## 2. 背景与原理介绍

当模型开始有参数以后，训练就不只是“算一个结果”，而是“算结果，再算结果对参数的影响”。这个“影响”就是梯度。

自动求导框架之所以强大，是因为它们把复杂计算拆成很多局部算子，每个算子只需要知道自己的前向和局部导数，整条链路的梯度就能通过链式法则自动回传。这就是反向传播的思想来源。

与此同时，真实训练不会一次读完整个数据集，而是把样本切成一个个小批次。这就是 mini-batch，它兼顾了效率、稳定性和内存消耗。

### 步骤 3：复现“局部导数拼起来就是整体导数”

挑一个非常简单的复合函数：

`z = (x * y + b) ** 2`

请你做三件事：

1. 手推 `dz/dx`、`dz/dy`、`dz/db`
2. 写出数值梯度近似
3. 对比两者是否一致

这一步是本课最重要的核心练习。

### 步骤 4：自己解释计算图

把上面的函数拆成几个节点：

1. `q = x * y`
2. `r = q + b`
3. `z = r ** 2`

然后从后往前写出：

1. `dz/dr`
2. `dr/dq`
3. `dq/dx`

最后得到 `dz/dx = dz/dr * dr/dq * dq/dx`

如果你能亲手写出这一步，后面的反向传播就不再神秘。

### 步骤 5：做一个更像神经网络的复杂链式

上面的标量例子非常适合第一次理解链式法则，但它还不够像真正的神经网络。为了把“偏导数练习”自然过渡到后面的 MLP，本课建议你再做一个单样本、单层非线性网络的反向传播：

给定：

- 输入：`x ∈ R^3`
- 参数：`W ∈ R^(3x2)`，`b ∈ R^2`
- 目标：`y ∈ R^2`

定义：

1. `z = x @ W + b`
2. `h = tanh(z)`
3. `diff = h - y`
4. `L = 1/2 * sum(diff^2)`

这个例子很适合夹在第二课里，因为它同时满足三点：

1. 已经是“矩阵乘法 + 激活函数 + 损失”的结构了，和后面的神经网络非常接近。
2. 规模还不大，仍然可以手推，不会在第二课就把你压到公式堆里。
3. 它能让你第一次真正感受到：反向传播不是魔法，只是沿着计算图把局部梯度一层层乘回去。

你在这一小节至少要做下面几件事：

1. 手推 `dL/diff`、`dL/dh`、`dL/dz`
2. 写出 `dL/dW`、`dL/db`、`dL/dx`
3. 用数值梯度检查这三个结果

对应代码我已经补进：

- `practice/lesson02/complex_backprop.py`

建议你先自己推，再去运行脚本对照中间结果。
