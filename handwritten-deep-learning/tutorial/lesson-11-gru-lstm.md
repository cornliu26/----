# 第 11 课：GRU、LSTM 与现代序列模型

## 1. 本课定位

- 对应 D2L：
  - `chapter_recurrent-modern` 中的 GRU、LSTM、深层 RNN、双向 RNN、Seq2Seq 等相关章节
- 建议时长：3 到 5 小时
- 课时目标：
  - 理解门控机制为什么有用
  - 至少手写一个 `GRUCell` 或 `LSTMCell`
  - 对现代序列模型建立结构化认识

## 2. 背景与原理介绍

普通 RNN 最大的问题，是长期依赖难学。随着时间步拉长，梯度在反复相乘中容易衰减或爆炸。GRU 和 LSTM 的核心改进思路，就是引入门控，让模型学会：

1. 什么信息该保留
2. 什么信息该遗忘
3. 什么信息该写入新的状态

这类结构看上去比普通 RNN 复杂，但本质都是“让信息流更可控”。

## 3. 详细操作过程

### 步骤 1：先比较结构图

请把普通 RNN、GRU、LSTM 的更新公式放在一起看，重点问自己三个问题：

1. 新状态是怎么生成的？
2. 旧状态如何保留？
3. 门控到底在控制哪一条信息通路？

### 步骤 2：先选一个单元实现

推荐优先选：

1. `GRUCell`，公式相对更紧凑
2. 如果你想挑战，再做 `LSTMCell`

不要试图同一晚同时手写两种，容易把公式搅在一起。

### 步骤 3：实现前向传播

实现单元时，先只处理单时间步。你需要明确：

1. 输入 `x_t`
2. 上一时刻状态
3. 当前时刻状态

如果做 LSTM，还要区分：

1. 隐状态 `h_t`
2. 细胞状态 `c_t`

### 步骤 4：把单元接到时间循环里

当单步单元正确后，再外层写一个时间循环，把整个序列跑完。建议先不自己写复杂训练逻辑，先保证前向结果和维度正确。

### 步骤 5：和普通 RNN 做一次对比实验

拿同一个小数据集，比较：

1. RNN
2. GRU
3. LSTM

重点不一定是绝对指标，而是：

1. 收敛速度
2. 训练稳定性
3. 长序列建模能力

### 步骤 6：理解 Seq2Seq 的角色

在这一课结尾，你不必完整实现大模型，但应该理解：

1. 编码器把输入序列压缩成状态
2. 解码器根据状态逐步生成输出
3. 这就是很多翻译和生成任务的早期基本套路

## 4. 本课检查点

1. GRU 和 LSTM 分别解决了普通 RNN 的什么问题？
2. 门控为什么可以改善长期依赖学习？
3. LSTM 中 `c_t` 和 `h_t` 的角色有什么不同？
4. Seq2Seq 为什么会引出后面的注意力机制？

2. 选做：继续实现 `LSTMCell`。
3. 写一页对比笔记，总结 RNN、GRU、LSTM 的区别。
