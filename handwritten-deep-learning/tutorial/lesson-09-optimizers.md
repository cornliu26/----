## 2. 背景与原理介绍

优化器本质上回答一个问题：知道了梯度以后，参数应该怎么更新？

最基础的 SGD 只看当前梯度，但现实中的训练曲面往往又陡又扭曲，直接按当前梯度走并不总是高效。于是就有了：

1. Momentum：让更新带一点“惯性”
2. AdaGrad：对不同参数自适应调整学习率
3. RMSProp：改进 AdaGrad 后期学习率过小的问题
4. Adam：把 Momentum 和自适应学习率思路结合起来

## 3. 详细操作过程

### 步骤 1：先建立统一优化器接口

在 `practice/mydl/optim.py` 里统一接口，建议至少有：

1. `zero_grad()`
2. `step()`

即使你现在的梯度不是自动累积的，这个接口也值得保留。

### 步骤 2：实现 SGD 基线版本

先把最简单的更新规则整理清楚：

`param = param - lr * grad`

把它作为所有后续实验的对照组。

### 步骤 3：加入 Momentum

为每个参数维护一个速度变量。关键问题是：

1. 为什么需要历史梯度的指数加权平均？
2. 为什么这有助于沿主要下降方向更平滑地前进？

建议用小实验观察 loss 曲线是否更平滑。

### 步骤 4：加入 Adam

Adam 需要为每个参数维护：

1. 一阶矩估计
2. 二阶矩估计
3. 偏差修正

这部分公式看上去多，但只要代码结构清楚，其实并不难。

### 步骤 5：做控制变量实验

拿同一个模型、同一个数据集，固定：

1. 随机种子
2. 初始参数
3. 训练轮数
4. 批大小

只更换优化器。记录：

1. loss 曲线
2. accuracy 曲线
3. 收敛速度

### 步骤 6：解释实验结果

请不要只写“Adam 更好”。你应该尝试解释：

1. 为什么它在你的实验里更快
2. 有没有更容易过拟合
3. 什么时候简单 SGD 反而足够甚至更稳

## 4. 本课检查点

1. SGD、Momentum、Adam 的核心差异是什么？
2. 为什么 Adam 需要偏差修正？
3. 优化器更快收敛，是否一定意味着泛化更好？
4. 你的实验是不是做到了公平对比？
