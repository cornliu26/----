  - 感受初始化和数值稳定性对训练的影响

## 2. 背景与原理介绍

模型能学会不代表模型学得对。很多时候，训练集效果很好，测试集却很差，这就是过拟合。还有些时候，模型连训练集都学不好，这叫欠拟合。

为了解决这些问题，深度学习里有三类非常常见的手段：

1. 正则化：约束模型不要太复杂
2. Dropout：训练时随机丢掉部分神经元，降低共适应
3. 初始化与数值稳定处理：避免梯度爆炸、消失和训练发散

## 3. 详细操作过程

### 步骤 1：先故意制造过拟合

拿上一课的 MLP 做实验：

1. 把训练集变小
2. 把隐藏层变大
3. 训练更多轮

然后同时画训练和验证曲线。你应该能看到两条曲线开始分叉。

### 步骤 2：实现 L2 正则化

在损失函数中加入：

`loss_total = loss_data + lambda * sum(W^2)`

实际编码时要注意：

1. 通常只对权重做正则，不对偏置做
2. 梯度中会多出与 `W` 成正比的一项

### 步骤 3：实现 Dropout

先理解训练和测试的区别：

1. 训练时，随机把部分激活置 0
2. 测试时，不再随机丢弃
3. 为了保持期望一致，训练时通常要做缩放

你可以先只对隐藏层加 Dropout，不要一上来加很多层。

### 步骤 4：比较不同初始化

至少比较三种初始化方式：

1. 全 0 初始化
2. 很小随机数初始化
3. 基于输入输出维度缩放的初始化

记录每种初始化下：

1. 前几轮 loss
2. 梯度大小
3. 是否出现不收敛

### 步骤 5：定位数值不稳定问题

如果训练中出现：

1. `nan`
2. loss 突然爆炸
3. accuracy 长时间不变

请依次检查：

1. 学习率
2. Softmax 是否做了稳定化处理
3. Dropout 是否只在训练模式生效
4. 初始化是否过大

## 4. 本课检查点

1. 过拟合和欠拟合在曲线上分别长什么样？
2. L2 正则为什么会让权重变小？
3. Dropout 为什么只在训练时使用？
4. 全 0 初始化为什么对多层网络不合适？

## 5. 课后小作业

1. 用表格对比“无正则、L2、Dropout、L2+Dropout”的结果。
2. 改变 Dropout 比例，比较模型性能。
3. 尝试加入 early stopping，并说明它和正则化的关系。
